# 樱花地图 UI 设计语言（固定规则）

本文件用于**固定**樱花地图的界面设计语言，避免组件各自为政。后续 UI 调整应优先对齐本规则；如需变更设计语言，请先更新本文件再改代码。

## 核心目标

- **无边框**：避免使用 `border` 产生明显描边
- **扁平化**：避免厚重投影与拟物层次，主要依靠背景色/间距/字号建立信息层级

## 视觉分层（推荐模式）

> 下面给出几种在代码中可直接复用的 Tailwind class 组合，用于保持一致性。

- **浮层容器（Popover/Dropdown）**：允许使用极轻的“轮廓型”阴影作为边界感（非 `border`）
  - 参考：`bg-popover ... shadow-lg shadow-[0_0_0_1px_rgba(0,0,0,0.06)]`
- **面板分组（Panel）**：用轻背景表达分组，不要描边
  - 推荐：`rounded-xl bg-muted/30 px-3 py-2.5`
- **面板内子块（Block）**：子块同样不描边，不加投影；用更浅/半透明背景做层级
  - 推荐：`rounded-lg bg-background/60 px-2 py-2`

## 组件规则

- **不要使用 `border border-border`** 来做卡片/分组分隔（尤其是 Popover 内部的嵌套卡片）。
- **Outline 视觉**（例如按钮）如需强调边界，优先使用“内描边 shadow”而不是 `border`：
  - 例如按钮 `variant="outline"` 使用的 `shadow-[inset_0_0_0_1px_rgba(0,0,0,0.08)]`
- **阴影**：避免 `shadow-md` 这类“浮起”的卡片投影；允许用于浮层的轻量轮廓 shadow（见上）。
- **图标**：统一使用 `lucide-react`；集合标签使用 `Award` 图标（不要使用 `Ribbon`）。

## 具体落点

- 花期预测面板：应按“Panel + Block”模式实现（无边框、扁平化），不要再嵌套带描边卡片。
